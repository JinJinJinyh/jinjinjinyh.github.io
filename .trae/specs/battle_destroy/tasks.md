
# 战地破坏 - 实施计划

## [/] 任务1：Unreal Engine项目设置
- **优先级**：P0
- **依赖**：无
- **描述**：
  - 创建Unreal Engine 5项目
  - 启用Chaos Physics
  - 设置基本的输入映射（移动、跳跃、射击、切换武器）
- **验收标准对应**：AC-4
- **测试要求**：
  - human-judgment：项目可以正常启动，玩家可以移动
- **备注**：使用第一人称模板

## [ ] 任务2：方块系统核心
- **优先级**：P0
- **依赖**：任务1
- **描述**：
  - 创建Block Actor基类
  - 实现生命值系统（最大60点）
  - 实现损坏阶段贴图切换（0-19完好，20-39轻微损坏，40-59严重损坏，60+摧毁）
  - 实现不同材质的方块（木头、石头、混凝土、金属）
- **验收标准对应**：AC-1
- **测试要求**：
  - human-judgment：方块在不同生命值阶段显示不同贴图
- **备注**：方块尺寸建议50cm

## [ ] 任务3：世界生成与网格系统
- **优先级**：P0
- **依赖**：任务2
- **描述**：
  - 创建Block Grid管理系统
  - 实现高效的方块位置查找
  - 创建基础的小地图
  - 放置一些可破坏的建筑物
- **验收标准对应**：AC-1, AC-4
- **测试要求**：
  - human-judgment：地图加载正常，建筑物由方块组成

## [ ] 任务4：武器系统 - 突击步枪
- **优先级**：P0
- **依赖**：任务3
- **描述**：
  - 创建武器基类
  - 实现突击步枪（自动射击，每发1点伤害）
  - 实现射线检测命中检测
  - 添加后坐力、射击效果
- **验收标准对应**：AC-1
- **测试要求**：
  - human-judgment：射击可以对方块造成伤害，连续射击60次可以摧毁一个方块

## [ ] 任务5：武器系统 - RPG
- **优先级**：P1
- **依赖**：任务4
- **描述**：
  - 实现RPG武器（ projectile-based）
  - 实现范围破坏（3x3x3方块）
  - 添加爆炸效果
- **验收标准对应**：AC-2
- **测试要求**：
  - human-judgment：RPG可以一次摧毁3x3x3范围内的方块

## [ ] 任务6：结构稳定性系统
- **优先级**：P0
- **依赖**：任务3
- **描述**：
  - 实现连通性检测算法（检查方块是否连接到地面）
  - 当方块被破坏时，触发稳定性检查
  - 将不稳定的方块转换为物理模拟的碎片
- **验收标准对应**：AC-3
- **测试要求**：
  - human-judgment：破坏承重墙后，建筑物会倒塌

## [ ] 任务7：物理碎片系统
- **优先级**：P1
- **依赖**：任务6
- **描述**：
  - 创建碎片Actor
  - 应用Chaos Physics物理模拟
  - 碎片碰撞时对其他方块造成伤害
  - 实现碎片的自动清理（一段时间后消失）
- **验收标准对应**：AC-3
- **测试要求**：
  - human-judgment：倒塌的方块会变成碎片，碎片有物理效果

## [ ] 任务8：整合与优化
- **优先级**：P1
- **依赖**：任务5, 7
- **描述**：
  - 整合所有系统
  - 性能优化（对象池、LOD等）
  - 打磨游戏感觉
- **验收标准对应**：AC-4
- **测试要求**：
  - human-judgment：游戏可以流畅运行，所有功能正常工作
